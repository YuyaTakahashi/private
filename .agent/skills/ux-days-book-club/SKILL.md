# UX Days Tokyo Book Club Report Skill

## 1. 概要 (Overview)
UX DAYS TOKYOで毎月開催される読書会のレポート記事を作成します。単なる「要約」ではなく、**「参加者がどう感じたか」「実務にどう活かせるか」**という視点を重視します。

## 2. ターゲット読者 (Target Audience)
- **メイン**: UXデザイナー、リサーチャー、PdM
- **属性**:
    - UX DAYS TOKYOの活動に興味がある人
    - 読書会に参加できなかったが、内容を知りたい人
    - 実務でUX/デザインの課題を抱えている人

## 3. 記事の構成 (Structure)
`writing-standard`スキルの「記事構成フレームワーク」に準拠し、以下の要素を含めます。

### 基本構造
**【結】→【起】→【承】→【転】→【結】** の流れを採用します。

### 1. 【結】導入・結論 (Conclusion First)
- **挨拶**: 「こんにちは。スタッフの〇〇です」
- **結論（一言）**: 今回の最も重要な学びを冒頭で提示する。
- **書籍紹介**: 書名、著者、開催日。

### 2. 【起】問題提起 (Problem)
- **動機**: なぜこの本を選んだのか？ 参加者が抱えていた課題や疑問。
- **文脈**: 世の中の潮流や、現場での悩み。

### 3. 【承】解決の方向性 (Direction)
- **学びの核**: 書籍から得られた主要なインサイト（3つ程度）。
- **引用**: 印象的なフレーズや図解への言及。

### 4. 【転】ディスカッション・実務適用 (Discussion & Application)
- **参加者の声**: 読書会で盛り上がったポイント、他の参加者の意見。
- **自分ごと化**: 自分の実務（PdM/デザイン）への適用アイデア。

### 5. 【結】まとめ (Conclusion & Action)
- **総括**: 学びのまとめ。
- **Next Step**: 明日から試せる具体的なアクションや、次回参加への呼びかけ。


### リンク・参照
- 関連するUX用語解説記事へのリンク（内部リンク）を意識的に配置する。
- 著者の他の書籍や、過去の読書会レポートへの言及。

## 4. トーン＆マナー (Tone & Manner)
- **文体**: "です・ます"調。親しみやすく、かつ知的なトーン。
- **視点**: スタッフ（高橋祐哉）としての個人的な所感を含める（`personal-context`スキルを参照）。
- **強調**: 重要な概念や書籍の引用は太字にする。

## 5. 入力情報 (Input)
- 書籍名
- 読書会のメモ、ログ、または録音の要約
- ユーザーが特に印象に残ったポイント（箇条書きでOK）
- (あれば) 参加者の具体的な発言内容

## 6. 生成プロセス (Process)
1.  **コンセプト設計**: `writing-standard`に基づき、「一言」「ターゲット価値」「構成案」を定義する。
2.  **入力の分析**: メモからコアとなる学びとエピソードを抽出する。
3.  **ドラフト作成**: 定義した構成に従って執筆する。
4.  **リンク提案**: UX Times用語集などへのリンクを検討する。

## 7. 執筆のコツ (Tips)
- **「自分ごと」化**: 単に本の内容を紹介するだけでなく、「私がどう変わったか」「現場でどう感じたか」を書くことで、読者の共感を呼ぶ。
- **具体的アクション**: 読み手が明日から使える視点やアクションを含める。
