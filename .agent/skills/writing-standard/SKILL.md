# Writing Standard (Article Creation Guide)

この記事作成ガイドは、単に文章を書くのではなく、**「誰に」「何を」「どう伝えるか」**を戦略的に設計するための共通ルールです。

## 1. コンセプト設計 (Concept Design)
執筆を開始する前に、必ず以下の3要素を言語化してください。

### ① 一言（いちごん / The One Thing）
記事全体で伝えたい最も重要なメッセージを「一言」で表現する。
- **ポイント**: 読んだ後に読者の心に残るコアメッセージ。
- **例**: 「UXerは誰にでもなれるよ」「どんな手法を使うにもユーザー視点が必要！」

### ② ターゲットへの提供価値 (Value Proposition)
具体的な読者像（ペルソナ）を定義し、読了後の状態（ゴール）を設定する。
- **自問**: この記事を読んだターゲットは、何を「読んで良かった」と感じるか？
- **例**: 「UXerってなんだ？」と悩む人に対し、「デザイナーだけでなく、組織全員でやるものだ」という気づきを与える。

### ③ 構成戦略 (Structure Strategy)
ビジネスライティングに適した「結論先出し」の起承転結フレームワークを採用する。
- **基本構成**: **【結】→【起】→【承】→【転】→【結】**

## 2. 記事構成フレームワーク (Structure Framework)

各パートの役割と内容は以下の通りです。

### 【結】最初に、結論 (Conclusion First)
- **内容**: 結論の前提・背景、結論そのもの、社会や事業との関わり。
- **目的**: 読者が冒頭で「この記事は何について書かれていて、何が言いたいのか」を把握させる。

### 【起】問題提起 (Introduction / Problem)
- **内容**: 読者が抱えている課題や疑問を提示し、共感を呼ぶ具体例を挙げる。
- **例**: 「本音を考えずに、思わず見てしまう記事は作れないですよね？」

### 【承】解決の方向性 (Direction / Insight)
- **内容**: 「どうすれば良いのか」という解決の方向性や、思考のヒントを提示する。
- **テクニック**: 権威ある言葉（マザー・テレサなど）や、ハッとするような引用を用いてメッセージを補強する。

### 【転】具体的な説明 (Body / Details)
- **内容**: 具体的な手法、スキル、役割、アクションについて詳述する。
- **例**: 具体的なコンサルテーション手法や、明日から使えるフレームワークの紹介。

### 【結】まとめ・ネクストアクション (Conclusion & Action)
- **内容**: 全体のまとめと、読者が「やってみよう」と思える行動。
- **注意**: 単なるやり方の紹介で終わらせず、「身近な人の言葉から思考を考えてみよう」など、具体的な行動変容を促す。

## 3. 執筆・表現ガイドライン (Writing Guidelines)

### ポジティブな影響を与える (Keep it Positive)
- **ネガティブ表現は避ける**:
    - × 「成長できていませんでした」「的外れな注文」
    - ○ 「〜という課題に気づくことができました」「より本質的な提案が必要でした」
- **「学び」を明確にする**:
    - × 「たくさんのケースで学んでいきました」（具体性がない）
    - ○ 「これらを理解することで、考え方の棲み分けができるようになりました」（メリットが明確）

### 図解のルール (Visual Rules)
- **シンプルに**: 「伝えるために必要なもの」以外は描かない。装飾や不要なテキストは排除する。
- **疑問を抱かせない**: パッと見て意味がわからない矢印や配置を避ける。
- **キャプション**:
    - × 「〇〇の図」（事実の説明）
    - ○ **「〇〇することで、××が解決する」**（その図で言いたいメッセージ）

## 4. プロセス (How to Use)
このスキルを使用する際は、以下のステップで出力を生成してください。

1.  **設計**: 入力情報から「一言」「ターゲット価値」「構成案（結・起承転結）」を出力する。
2.  **執筆**: 設計に基づき、本文を執筆する。
3.  **推敲**: ガイドライン（ネガティブ表現の排除など）に基づいて自己レビューを行う。
